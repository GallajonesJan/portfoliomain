TaskFlow

Full-Stack Task Manager with Real-Time Collaboration and AI-Assisted Development

Project Overview

TaskFlow is a full-stack task management web application designed for both personal and professional use. The project covers the complete software development lifecycle from UI design and frontend implementation to backend API development, database integration, and cloud deployment.

Built with React on the frontend and Node.js with Express on the backend, TaskFlow uses PostgreSQL hosted on Supabase as its database. The system supports user authentication, task assignments, real-time notifications, file attachments, comments, and a fully responsive interface with light and dark mode.

System Architecture

TaskFlow follows a three-tier architecture with a clear separation between the frontend, backend API, and the Supabase-hosted database and auth layer.

Frontend (React + Vite)

  • React SPA: Component-based UI with React Router for page navigation
  • State Management: Local state with useState and useCallback; centralized API calls via a service layer
  • SCSS Modules: Per-component SCSS files using a shared variables system for consistent theming
  • Real-Time Updates: Supabase Realtime channels for live notification delivery
  • Auth Context: Global user session managed via React Context with Supabase Auth

Backend (Node.js + Express)

  • RESTful API: Modular route files for tasks, categories, users, assignments, comments, and attachments
  • Admin Middleware: Header-based admin check using a configurable ADMIN_EMAIL environment variable
  • File Uploads: Multer middleware for parsing multipart form data; files stored in Supabase Storage buckets
  • Email Notifications: Nodemailer with Gmail SMTP for comment and assignment alerts
  • Preflight Checks: Server startup validates all environment variables and Supabase connectivity before accepting traffic

Database (PostgreSQL via Supabase)

  • Tables: tasks, categories, profiles, task_assignments, task_comments, task_attachments, notifications
  • Row Level Security: Supabase RLS policies restrict data access per user role
  • Triggers: Auto-update timestamps on comment edits via PostgreSQL triggers
  • Auth Integration: Supabase Auth manages user sessions; profiles table extends auth.users

Technical Challenges & Solutions

Challenge 1: Role-Based Access Without a Full Auth Server
Solution: Used Supabase Auth for session management combined with a custom ADMIN_EMAIL environment variable on the backend. The frontend injects an x-user-email header with every API request, which the backend middleware checks against the configured admin email to grant or restrict access.

Challenge 2: Real-Time Notifications Across Sessions
Solution: Leveraged Supabase Realtime's postgres_changes channel to subscribe to INSERT events on the notifications table filtered by user ID. New notifications appear instantly in the bell icon without polling.

Challenge 3: File Uploads Through a Backend Proxy
Solution: Configured Multer on the Express backend to accept multipart file data, then forwarded the file buffer directly to Supabase Storage using the service role key. This keeps storage credentials server-side and out of the browser.
